# Hyper-RAG · Epistemic Knowledge Hypergraph

A Python + interactive HTML implementation of hypergraph-driven retrieval-augmented
generation with epistemic evidence scoring.

Retrieval follows the three-stage algorithm from:
> Feng et al. (2025) *Hyper-RAG: Combating LLM Hallucinations using Hypergraph-Driven
> Retrieval-Augmented Generation* — arXiv:2504.08758

The epistemic scoring layer (9 dimensions, quality gate) is an original extension
on top of the paper's retrieval mechanism.

---

## How the retrieval actually works

Standard RAG converts every document to a vector and ranks by cosine similarity — the
graph structure is never used. Hyper-RAG's three stages change this:

```
Stage 1 — Entity extraction
  Match query tokens against known node ids/labels in the hypergraph.
  e.g. "How does BDNF affect memory?" → ['BDNF', 'episodic_memory', 'spatial_memory', ...]

Stage 2 — Hypergraph traversal  (the core difference from standard/graph RAG)
  Seed:  find all hyperedges containing ≥1 query entity
  1-hop: expand through ALL nodes co-occurring in seed edges
  2-hop: expand again through nodes reached at 1-hop

  Why hyperedges beat binary edges here:
  A hyperedge connects N entities simultaneously, so touching one seed node
  immediately surfaces all N co-members. One hop covers far more ground than
  expanding pairwise edges would — and surfaces cross-cluster connections
  (e.g. a microglia/TREM2 query reaching BDNF evidence at 1-hop via
  shared hippocampus/memory nodes).

Stage 3 — Scoring
  final_score = 0.35 · entity_coverage
              + 0.35 · semantic_similarity   (TF-IDF cosine)
              + 0.30 · epistemic_quality      (our 9-dimension score)
```

A flat TF-IDF baseline (`retrieve_flat`) is kept for direct comparison.

## File map

```
hyper_rag_project/
├── main.py          ← SINGLE ENTRY POINT — run this
├── hypergraph.py    ← data structures (EvidenceType, EpistemicScore, Node, Hyperedge, KnowledgeHypergraph)
├── corpus.py        ← 14 curated neuroscience passages across 4 topic clusters
├── builder.py       ← builds the hypergraph; derives epistemic scores from passage metadata
├── retrieval.py     ← Hyper-RAG engine: entity extraction → traversal → 3-way scoring
├── demo.py          ← 4 terminal demos showing traversal working
├── viz_export.py    ← serialises hypergraph to JSON for the browser visualisation
└── viz/
    ├── index.html   ← interactive D3 visualisation (self-contained, no server needed)
    └── hypergraph_data.json  ← auto-generated by main.py
```

**Module dependencies:**

```
corpus.py  →  builder.py  →  KnowledgeHypergraph
                                    ↓              ↓
                             retrieval.py     viz_export.py
                                    ↓              ↓
                              demo.py        viz/index.html
                                    ↑
                              main.py  (orchestrates all)
```

```bash
pip install scikit-learn rich numpy
```

```bash

python main.py --query "How does sleep affect hippocampal memory?"
python main.py --query "dopamine reward" --top-k 3
```

## Interactive visualisation (`viz/index.html`)

- **Squares** = hyperedges; colour = evidence type; brightness = epistemic quality
- **Circles** = entity nodes; size = degree (how many hyperedges contain this entity)
- **Click any square** → right panel shows the 9-dimension epistemic breakdown
- **α slider** → adjust relevance vs. epistemic weighting live
- **Quality gate slider** → fade out edges below a minimum epistemic score
- **Evidence type legend** → toggle types on/off
- Drag, scroll to zoom, "Reset view" / "Reheat layout"

## The 9 epistemic dimensions (`hypergraph.py → EpistemicScore`)

These dimensions come from the research proposal; the weights are
placeholders pending formal derivation or expert elicitation.

| Dimension | Weight | What it captures |
|-----------|--------|-----------------|
| Methodological strength | 0.20 | EvidenceType rank (meta-analysis → commentary) |
| Directness | 0.15 | Does evidence address the claim directly? |
| Source authority | 0.10 | Journal prestige heuristic |
| Specificity | 0.10 | Exact conditions vs. general claim |
| Sample adequacy | 0.10 | Log-scaled sample size |
| Independence | 0.10 | Are corroborating sources genuinely independent? |
| Consistency | 0.10 | Agreement vs. contradiction with consensus |
| Replication status | 0.10 | How often reproduced |
| Recency | 0.05 | Age decay from current year |

Aggregate: `Σ weight_i × score_i` (weighted linear, all in [0, 1])

## Corpus (`corpus.py`)

14 neuroscience passages across four topic clusters, each with full metadata
(source journal, evidence type, year, sample size, entity list) used by
`builder.py` to auto-derive all 9 epistemic dimensions.

- **Cluster A** — BDNF, TrkB, LTP, hippocampus, spatial/episodic memory
- **Cluster B** — Dopamine, VTA, reward prediction error, decision making
- **Cluster C** — Microglia, TREM2, amyloid, neuroinflammation, Alzheimer's
- **Cluster D** — Sleep, sharp-wave ripples, hippocampal memory consolidation
